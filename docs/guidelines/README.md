# Guidelines

Guidelines record reusable mechanics. They do not replace current facts in
`ARCHITECTURE.md` or unit behavior in `docs/specs/`.

- [agent-workflow.md](agent-workflow.md) - repo reading order and agent change
  discipline.
- [testing-harness.md](testing-harness.md) - backend, Go, and frontend test
  mechanics.
- [frontend-contracts.md](frontend-contracts.md) - OpenAPI aggregation and
  orval client generation.
- [spring-platform.md](spring-platform.md) - Spring Boot, RestClient, events,
  resilience, and Spring AI mechanics.
- [course-deliverables.md](course-deliverables.md) - how to treat SRS, SDD, and
  diagram outputs.

Older reference files such as `docs/gotchas.md`, `docs/testing.md`,
`docs/observability.md`, and topical `docs/*.html` remain available, but new
durable mechanics should be added here.
