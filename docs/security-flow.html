1. Kiến trúc tổng quan

Auth-service generate RSA keypair ở startup, publish public keys qua /.well-known/jwks.json. Mọi service downstream (customers/vets/gateway) tự fetch JWKS + cache + validate RS256 signature → zero-trust defense-in-depth. Refresh tokens lưu DB dưới dạng SHA-256 hash, rotate single-use, detect reuse → revoke all user tokens. Gateway thêm per-IP rate limit cho auth endpoints + circuit breaker cho mọi route.

2. RSA keypair + JWKS publishing

Auth-service generate RSA-2048 keypair tại startup (kid = random UUID). Private key giữ trong memory, public key publish qua endpoint chuẩn OIDC.

services/auth-service/.../security/RsaKeyConfig.java
@Configuration
public class RsaKeyConfig {

    @Bean
    public RSAKey rsaJwk() throws NoSuchAlgorithmException {
        KeyPairGenerator kpg = KeyPairGenerator.getInstance("RSA");
        kpg.initialize(2048);
        var kp = kpg.generateKeyPair();
        return new RSAKey.Builder((RSAPublicKey) kp.getPublic())
                .privateKey((RSAPrivateKey) kp.getPrivate())
                .keyID(UUID.randomUUID().toString())
                .build();
    }

    @Bean public JWKSet jwkSet(RSAKey rsaJwk) { return new JWKSet(rsaJwk); }
    @Bean public JWKSource<SecurityContext> jwkSource(JWKSet jwkSet) {
        return new ImmutableJWKSet<>(jwkSet);
    }
}
services/auth-service/.../controller/JwksController.java
@RestController
public class JwksController {
    private final JWKSet jwkSet;
    public JwksController(JWKSet jwkSet) { this.jwkSet = jwkSet; }

    @GetMapping(value = "/.well-known/jwks.json", produces = APPLICATION_JSON_VALUE)
    public Map<String, Object> jwks() {
        return jwkSet.toJSONObject(true);   // true = PUBLIC keys only (strip private d, p, q, ...)
    }
}

shared/common-security/.../autoconfigure/PetClinicSecurityAutoConfiguration.java
@AutoConfiguration
@EnableConfigurationProperties(PetClinicJwtProperties.class)
public class PetClinicSecurityAutoConfiguration {

    @Bean
    @ConditionalOnMissingBean
    public JwtDecoder jwtDecoder(PetClinicJwtProperties props) {
        NimbusJwtDecoder decoder = NimbusJwtDecoder
                .withJwkSetUri(props.getJwkSetUri()).build();
        decoder.setJwtValidator(JwtValidators.createDefaultWithValidators(
                new JwtTimestampValidator(),
                new JwtClaimValidator<String>("iss", props.getIssuer()::equals),
                new JwtClaimValidator<List<String>>("aud",
                        aud -> aud != null && aud.contains(props.getAudience()))
        ));
        return decoder;
    }

    @Bean
    @ConditionalOnMissingBean
    public JwtAuthenticationConverter jwtAuthenticationConverter() {
        JwtGrantedAuthoritiesConverter granted = new JwtGrantedAuthoritiesConverter();
        granted.setAuthoritiesClaimName("roles");
        granted.setAuthorityPrefix("ROLE_");
        JwtAuthenticationConverter conv = new JwtAuthenticationConverter();
        conv.setJwtGrantedAuthoritiesConverter(granted);
        conv.setPrincipalClaimName("sub");
        return conv;
    }
}

Cấu hình: config-repo/application-dev.yml đặt petclinic.auth.jwt.jwk-set-uri: http://localhost:8183/.well-known/jwks.json. Nimbus cache theo HTTP Cache-Control header (default 5 phút).

services/auth-service/.../dto/req/RegisterRequest.java
public record RegisterRequest(
        @NotBlank @Size(min = 3, max = 100) String username,
        @NotBlank @Email @Size(max = 100) String email,
        @NotBlank @Size(min = 8, max = 100) String password
) {
    public User toEntity(PasswordEncoder passwordEncoder) {
        User u = new User();
        u.setUsername(username);
        u.setEmail(email);
        u.setPassword(passwordEncoder.encode(password));     // BCrypt 12 rounds
        u.setRoles(new HashSet<>(Set.of("USER")));            // default role
        u.setEnabled(true);
        return u;
    }
}
Decision: Register KHÔNG auto-return JWT. Client phải gọi /login sau. Tách rõ register vs login để sau này (iter sau) thêm email verification mà không phải break API.

services/auth-service/.../security/JwtTokenProvider.java
public IssuedToken issueAccessToken(User user) {
    Instant now = Instant.now();
    JwtClaimsSet claims = JwtClaimsSet.builder()
            .issuer(jwtProps.getIssuer())
            .audience(List.of(jwtProps.getAudience()))
            .subject(user.getId().toString())
            .issuedAt(now)
            .expiresAt(now.plus(accessTokenTtl))
            .claim("username", user.getUsername())
            .claim("roles", user.getRoles())
            .build();
    JwsHeader header = JwsHeader.with(SignatureAlgorithm.RS256)
            .keyId(rsaJwk.getKeyID())         // services lookup public key qua kid
            .build();
    String token = jwtEncoder.encode(JwtEncoderParameters.from(header, claims)).getTokenValue();
    return new IssuedToken(token, accessTokenTtl.toSeconds());
}
services/auth-service/.../security/RefreshTokenService.java (excerpt)
@Transactional
public IssuedRefresh issue(UUID userId) {
    String raw = generateRaw();                       // SecureRandom 256-bit hex
    String hash = sha256(raw);
    RefreshToken token = new RefreshToken();
    token.setUserId(userId);
    token.setTokenHash(hash);                         // store hash, not raw
    token.setExpiresAt(Instant.now().plus(ttl));      // 7 days
    repo.save(token);
    return new IssuedRefresh(raw, ttl.toSeconds());
}

5. Authenticated request — Defense-in-depth

Gateway validate JWT lớp 1, downstream service validate LẠI lớp 2. Cùng share shared/common-security module → cùng JWKS URI, cùng decoder logic.

sequenceDiagram autonumber actor U as Client participant GW as gateway :8080 participant CUST as customers-service :8181 participant JWKS as auth-service JWKS U->>GW: GET /api/v1/owners
Authorization: Bearer eyJraWQi... rect rgb(22, 78, 99) Note over GW: Defense LỚP 1 GW->>GW: SecurityFilterChain decode JWT alt cache miss (kid mới) GW->>JWKS: GET /.well-known/jwks.json JWKS-->>GW: public RSA keys end GW->>GW: verify signature, check iss/aud/exp alt token hỏng GW-->>U: 401 (KHÔNG forward — kill sớm) end end GW->>CUST: forward request (Authorization header passthrough) rect rgb(120, 53, 15) Note over CUST: Defense LỚP 2 (zero-trust) CUST->>CUST: SAME SecurityFilterChain (common-security autoconfig) CUST->>JWKS: GET JWKS (nếu cache miss) CUST->>CUST: verify SAME signature độc lập CUST->>CUST: @PreAuthorize("hasRole('USER')") CUST-->>GW: 200 + data GW-->>U: 200 + data end
Tại sao 2 lớp? Không trust internal network. Attacker có thể:
  • Bypass gateway nếu khám phá pod IP trực tiếp (k8s misconfig)
  • Compromise gateway → inject malicious headers
  • Service mesh proxy bug
Cost: ~1ms/service/request (Nimbus verify cached public key cực nhanh).

Default SecurityFilterChain (apply cho customers + vets)

shared/common-security/.../autoconfigure/PetClinicSecurityAutoConfiguration.java
@Bean
@ConditionalOnMissingBean
public SecurityFilterChain defaultSecurityFilterChain(
        HttpSecurity http, JwtAuthenticationConverter jwtAuthConverter) throws Exception {
    http
        .csrf(AbstractHttpConfigurer::disable)
        .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
        .authorizeHttpRequests(auth -> auth
                .requestMatchers("/actuator/health/**", "/actuator/info",
                                 "/v3/api-docs/**", "/swagger-ui/**").permitAll()
                .anyRequest().authenticated()
        )
        .oauth2ResourceServer(oauth2 -> oauth2.jwt(jwt ->
                jwt.jwtAuthenticationConverter(jwtAuthConverter)));
    return http.build();
}

6. Refresh token rotation

sequenceDiagram autonumber actor U as Client participant AS as auth-service participant DB as Postgres Note over U: Access token hết hạn sau 15min U->>AS: POST /api/v1/auth/refresh
{refreshToken: "47cb3a93..."} AS->>AS: SHA-256(input) → hash AS->>DB: SELECT WHERE token_hash=? AND revoked=false AND expires_at>now alt hợp lệ AS->>DB: UPDATE old: revoked=true (single-use rotation) AS->>DB: INSERT new refresh_token AS->>AS: issue new access JWT AS-->>U: {accessToken (new), refreshToken (new), ...} else revoked = true (REUSE ATTACK detected) AS->>DB: revokeAllForUser(userId) — token đã dùng → kẻ tấn công đang dùng leak AS-->>U: 400 invalid-refresh-token else expired / not found AS-->>U: 400 invalid-refresh-token end
services/auth-service/.../security/RefreshTokenService.java
@Transactional
public UUID consumeAndValidate(String rawToken) {
    String hash = sha256(rawToken);
    RefreshToken existing = repo.findByTokenHash(hash).orElse(null);
    if (existing == null || !existing.isUsable(Instant.now())) {
        // REUSE DETECTION: nếu token đã revoked → revoke MỌI token của user này
        // Signal: original đã được dùng, attacker đang dùng leak
        if (existing != null && existing.isRevoked()) {
            repo.revokeAllForUser(existing.getUserId());
        }
        throw new InvalidRefreshTokenException();
    }
    existing.setRevoked(true);                        // single-use mark
    repo.save(existing);
    return existing.getUserId();
}

Tested ✓

Refresh với token hợp lệ → 200 + new access + new refresh
Refresh với token cũ (đã rotate) → 400 invalid-refresh-token + revoke ALL user tokens
Refresh với token mới (vừa nhận) → 200

7. Logout

Stateless JWT không "logout" thực sự — access token vẫn valid đến exp. Logout = revoke ALL refresh tokens của user → khi access token hết hạn, user không thể refresh nữa, phải đăng nhập lại.

services/auth-service/.../service/impl/AuthServiceImpl.java
@Override
@Transactional
public void logout(UUID userId) {
    refreshTokenService.revokeAllForUser(userId);
    audit.logoutSuccess(userId);
}

8. Rate limit ở gateway

Spring Cloud Gateway WebMVC 5.0.x KHÔNG có rate limit filter built-in (chỉ Reactive variant có). Tự viết HandlerFilterFunction dùng ConcurrentHashMap<ip, Bucket>. 10 req/phút cho /login, /register, /refresh — chống brute-force.

services/api-gateway/.../config/PerIpRateLimit.java (excerpt)
public HandlerFilterFunction<ServerResponse, ServerResponse> asFilter() {
    return (request, next) -> {
        String ip = request.servletRequest().getRemoteAddr();
        Bucket bucket = buckets.computeIfAbsent(ip, k -> new Bucket());
        if (!bucket.tryAcquire()) {
            return ServerResponse.status(TOO_MANY_REQUESTS)
                    .header("Retry-After", String.valueOf(periodMillis / 1000))
                    .body("""
                          {"type":".../rate-limited",
                           "title":"Rate limit exceeded",
                           "status":429,...}""");
        }
        return next.handle(request);
    };
}
services/api-gateway/.../config/GatewayRoutesConfig.java (auth routes only)
@Bean
public RouterFunction<ServerResponse> authPublicRoute() {
    PerIpRateLimit limiter = new PerIpRateLimit(10, Duration.ofMinutes(1));
    return route("auth-public")
            .route(path("/api/v1/auth/login")
                    .or(path("/api/v1/auth/register"))
                    .or(path("/api/v1/auth/refresh")), http())
            .filter(limiter.asFilter())                  // ← rate limit per-IP
            .filter(lb("auth-service"))                  // ← Eureka load-balanced
            .filter(circuitBreaker(c -> c.setId(CB_ID)
                    .setFallbackUri("forward:/fallback")))
            .build();
}

9. Audit log (ECS structured)

Mỗi auth event log structured theo Elastic Common Schema. MDC fields được structured-logging ECS format (Spring Boot 4) emit ra JSON line ở prod profile, Loki/Elasticsearch parse trực tiếp.

services/auth-service/.../security/AuthAuditLogger.java (excerpt)
private static final Logger log = LoggerFactory.getLogger("petclinic.auth.audit");

public void loginSuccess(UUID userId, String username) {
    try (MdcContext ctx = mdc("authentication.login.success", "success", userId, username)) {
        log.info("Login success");
    }
}

public void loginFailure(String username, String reason) {
    try (MdcContext ctx = mdc("authentication.login.failure", "failure", null, username)) {
        MDC.put("event.reason", reason);
        log.warn("Login failure");
        MDC.remove("event.reason");
    }
}
