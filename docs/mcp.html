petclinic-ms mcp-server expose 8 Tools (Customers/Vets/Visits domain), 2 Resources (FAQ + Specialty catalog), 2 Prompts (visit-summary + owner-contact-script) qua Spring AI 2.0.0-M6 + STATELESS Streamable HTTP transport. OAuth 2.1 Resource Server (JWT JWKS từ auth-service) + /.well-known/oauth-protected-resource metadata per RFC 9728. Annotation-driven: @McpTool / @McpResource / @McpPrompt auto-scan, không cần ToolCallbackProvider thủ công.

2.2. SEP-1442 — "Make MCP Stateless (by default)"

Slide #14-17 nhấn mạnh proposal SEP-1442: future direction là MCP stateless mặc định. petclinic-ms adopt từ đầu:

config-repo/mcp-server.yml
spring:
  ai:
    mcp:
      server:
        protocol: STATELESS              # ← SEP-1442 alignment
        name: petclinic-mcp-server
        version: 0.2.0
        type: SYNC                        # Java 25 virtual threads
        annotation-scanner:
          enabled: true                   # Auto-pickup @McpTool/@McpResource/@McpPrompt
Trade-off STATELESSDetail
✅ Load balancer trivialKhông cần sticky session, request rải random
✅ Scale ngang vô tưMỗi instance độc lập, no shared state
⚠️ KHÔNG sampling/elicitation từ serverCần bi-directional persistent connection. Petclinic chỉ expose tools/resources/prompts → OK

3. Authorization — MCP spec 2025-11-25 (slide #11-12)

Spring IO 2026 keynote dành 2 trang nguyên cho phần này. Spec mới (2025-11-25) định nghĩa MCP server = OAuth 2.1 Resource Server.

3.1. Flow

1. Client gọi /mcp không tokenMCP server trả 401 Unauthorized + header WWW-Authenticate: Bearer ...
2. Client fetch /.well-known/oauth-protected-resourceMCP server trả metadata: authorization_servers, scopes, bearer_methods
3. Client redirect user → authorization server OAuth flow(petclinic: auth-service JWT issue)
4. Client lấy access tokenBearer JWT signed bằng RSA, JWKS publish ở auth-service
5. Client gọi /mcp với Authorization: Bearer <jwt>MCP server verify qua JwtDecoder (JWKS) → OK / 401

3.2. Implementation

mcp-server/.../config/McpSecurityConfig.java
@Bean
public SecurityFilterChain mcpSecurityFilterChain(HttpSecurity http, JwtAuthenticationConverter conv) throws Exception {
    http
        .csrf(AbstractHttpConfigurer::disable)
        .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/.well-known/**", "/actuator/health/**", ...).permitAll()
            .requestMatchers("/mcp/**").authenticated()
            .anyRequest().authenticated())
        .oauth2ResourceServer(o -> o.jwt(j -> j.jwtAuthenticationConverter(conv)));
    return http.build();
}
mcp-server/.../config/WellKnownController.java
@GetMapping("/.well-known/oauth-protected-resource")  // RFC 9728
Map<String, Object> protectedResourceMetadata() {
    return Map.of(
        "resource", "http://localhost:8187/mcp",
        "authorization_servers", List.of("petclinic-ms"),   // issuer claim
        "bearer_methods_supported", List.of("header"),
        "scopes_supported", List.of("openid", "profile"),
        "resource_documentation", "...");
}

Lý do KHÔNG dùng spring-ai-community/mcp-security library: Version 0.1.11 advertise "Spring AI 2.0.x branch" nhưng chưa verify với 2.0.0-M6 milestone hiện tại (churn risk). Manual config = 0 dep mới, full control, pattern consistent với 4 service khác đã dùng common-security.
