# PCMS on Google Cloud (GKE)

Deployment manifests for running PCMS on GKE for the course demo. **Not yet
applied / verified — these are authored ahead of the actual deploy.**

## Topology (pro split: stateless on k8s, stateful + observability on VMs)

```
                       ┌──────────────────────── GKE (stateless apps) ────────────────────────┐
  Cloud Storage+CDN    │  config-server · discovery-server(Eureka) · admin-server             │
   (frontend SPA)      │  api-gateway · auth · customers · vets · visits · reviews · genai ·   │
        │              │  mcp · workflow · mailer(Go)        + Alloy DaemonSet (metrics/logs)  │
        ▼              └──────┬───────────────────────────┬───────────────────────────┬───────┘
   GKE Ingress (HTTPS LB) ───┘            JDBC/AMQP/cache/S3│              OTLP traces │  remote_write/loki
        │ -> api-gateway                                   ▼                          ▼
                                          ┌─ Data/Infra VM ──────────┐   ┌─ Observability VM ────────┐
                                          │ Postgres+pgvector        │   │ Grafana (UI)              │
                                          │ RabbitMQ · Redis · MinIO │   │ Prometheus · Loki · Tempo │
                                          │ Camunda + Elasticsearch  │   │ Alloy (OTLP gateway)      │
                                          │ Mailpit (dev SMTP)       │   │ postgres/redis exporters  │
                                          └──────────────────────────┘   └───────────────────────────┘
```

- **GKE** = stateless, cattle. Kustomize `base/` + `overlays/gke/`.
- **Data/Infra VM** = all stateful backends via `infra-vm/docker-compose.data.yml`.
- **Observability VM** = self-hosted LGTM via `infra-vm/docker-compose.observability.yml`
  (isolated from the cluster it monitors). Loki/Tempo can use the Data VM's MinIO as S3 backend.
- App services reach the VMs through selector-less Services in `base/external-services.yaml`
  (cluster and VMs share one VPC; lock ports with firewall rules).

## Layout

```
k8s/
├── base/                 # namespace, config, secrets(example), external-services,
│   │                     # alloy agent, ingress, + one Deployment+Service per service
│   └── kustomization.yaml
├── overlays/gke/         # Artifact Registry image rewrites + NEG patch
└── infra-vm/             # docker-compose for the two VMs
```

`base/*.yaml` per-service files are generated by `_generate.py` — edit the
generator, not the files.

## Deploy (when ready)

```bash
# 0. Prereqs: GKE cluster + 2 VMs in one VPC; firewall VM ports to the cluster CIDR.
#    Upgrade the billing account to PAID early to lift the free-trial vCPU quota.

# 1. VMs
gcloud compute scp infra-vm/* data-vm:~/   # then on the VM:
docker compose -f docker-compose.data.yml up -d
docker compose -f docker-compose.observability.yml up -d   # on the Obs VM

# 2. Build + push images to Artifact Registry (one per service)
gcloud builds submit --tag REGION-docker.pkg.dev/PROJECT_ID/petclinic/<svc> .

# 3. Fill placeholders
#    - base/external-services.yaml : VM internal IPs (10.10.0.10 / 10.10.0.20)
#    - overlays/gke/kustomization.yaml : REGION / PROJECT_ID
#    - copy base/secrets.example.yaml -> base/secrets.yaml, set real values
#    - base/ingress.yaml : host after the LB IP is known

# 4. Apply
gcloud container clusters get-credentials petclinic --zone ZONE
kubectl apply -k overlays/gke
kubectl -n petclinic get pods -w
```

## Notes / caveats

- **LLM:** keep `genai-service` pointed at OpenRouter/OpenAI (external). The free
  $300 credit cannot be spent on partner GenAI model-as-a-service.
- **Frontend** is hosted on Cloud Storage + CDN, not in the cluster.
- **Grafana** lives on the Obs VM — expose it via its own reverse proxy / firewall,
  not through this cluster Ingress.
- Tear everything down after the demo: `kubectl delete -k overlays/gke`,
  `docker compose down` on both VMs, delete the cluster + VMs.
